Notas para Certificação - Panoptes Patrol
Descrição da Aplicação
O Panoptes Patrol (nome de produto: PANOPTES PATROL) é uma aplicação de monitoramento de arquivos e diretórios que rastreia modificações, criações, exclusões, movimentações e renomeações de arquivos em tempo real. A aplicação utiliza a interface gráfica PySide6 (Qt) e possui recursos de visualização estatística e exportação de dados.

Ambiente de Teste:
Requisitos de Sistema:

- Windows 10 ou superior
- Python 3.8+ (se estiver executando a versão de código-fonte)
- Aproximadamente 150MB de espaço em disco para instalação completa

Componentes Principais:

- Interface gráfica baseada em PySide6/Qt
- Sistema de monitoramento utilizando API win32 (pywin32)
- Banco de dados SQLite para armazenamento local de eventos
- Visualização de estatísticas com matplotlib/seaborn

Instruções para Teste:

1. Inicialização e Configuração Básica:

	- Ao iniciar, o aplicativo exibe uma interface vazia aguardando seleção de diretório
	- Use o botão "Selecionar Diretório" para escolher uma pasta para monitorar
	- Use o botão "Iniciar/Parar" para ativar o monitoramento

2. Teste de Monitoramento (Funcionalidade Principal):

	1. Selecione um diretório de teste contendo pelo menos 5-10 arquivos
	2. Inicie o monitoramento
	3. Realize as seguintes operações no diretório monitorado:

		- Crie um novo arquivo de texto
		- Modifique o conteúdo de um arquivo existente
		- Renomeie um arquivo
		- Mova um arquivo para uma subpasta
		- Exclua um arquivo

	4. Verifique se todas as operações são detectadas e exibidas na tabela principal

3. Teste de Filtros e Visualização:

	1. Acesse "Configurações → Filtros"
	2. Teste os filtros por tipo de operação (active/desative diferentes tipos)
	3. Teste busca por texto no nome do arquivo
	4. Teste o filtro por extensão (por exemplo, filtre apenas arquivos .txt)
	5. Teste o filtro por data (selecione um período específico)

4. Teste de Exportação de Dados:

	1. Realize algumas operações de arquivo para gerar dados na tabela
	2. Clique em "Salvar Como" e teste os seguintes formatos:

		- Excel (.xlsx)
		- CSV (.csv)
		- JSON (.json)
		- XML (.xml)
		- SQLite (.db)

5. Teste de Estatísticas:

	1. Com pelo menos 10 eventos registrados, clique no botão "Estatísticas"
	2. Verifique cada um dos gráficos disponíveis
	3. Teste a funcionalidade "Salvar Todos" para exportar os gráficos

6. Teste de Idiomas:

	1. Acesse "Opções → Idioma"
	2. Mude para cada idioma disponível e verifique se a interface é traduzida corretamente

Comportamentos Esperados/Notas:

- Permissões de diretório: O aplicativo requer permissões de leitura no diretório monitorado
- Primeiro escaneamento: Ao iniciar o monitoramento, um escaneamento inicial será realizado (pode demorar alguns segundos para diretórios grandes)
- Eventos em lote: Operações em massa (como extrair um ZIP) podem gerar muitos eventos simultâneos
- Uso de recursos: O monitoramento de diretórios com muitos arquivos ou com mudanças frequentes pode consumir mais memória e CPU

Dependências Externas:

	A aplicação utiliza bibliotecas externas (todas incluídas no pacote), mas não depende de serviços NT ou drivers externos ao Windows. As principais dependências são bibliotecas Python, todas distribuídas com o aplicativo.

Pastas para Teste:

	Para melhor experiência de teste, recomenda-se criar uma pasta dedicada contendo:

	- Arquivos de texto (.txt, .md)
	- Imagens (.jpg, .png)
	- Documentos (.pdf, .docx)
	- Arquivos de mídia (.mp3, .mp4)

Instruções para Teste do Aplicativo de Monitoramento de Arquivos.

3. Teste de Filtros:

- Acesse "Configurações → Filtros"

- Gere eventos na tabela e exporte nos formatos:
    - Excel (.xlsx), CSV (.csv), JSON (.json), XML (.xml) e SQLite (.db)

5. Teste de Estatísticas:

- Com 10+ eventos registrados, verifique os gráficos disponíveis
- Teste função "Salvar Todos" para exportar gráficos

6. Teste de Idiomas:

- Em "Opções → Idioma", verifique se a interface é traduzida corretamente em cada idioma

Comportamentos Esperados:

- Aplicativo requer permissões de leitura no diretório
- Escaneamento inicial ocorre ao iniciar monitoramento
- Operações em massa podem gerar muitos eventos simultâneos
- Monitoramento de diretórios grandes pode consumir mais recursos
